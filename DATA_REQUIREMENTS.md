"""
Causal Agent — 数据需求与训练方案
=================================

因果推断智能体和传统 ML 模型有本质区别：
它不需要海量标注数据来「学习模式」，
它需要的是 —— 因果结构知识 + 适度规模的观测数据。

═══════════════════════════════════════════════════════════════
一、核心洞察：结构与数据的分离
═══════════════════════════════════════════════════════════════

传统 ML:  Data → Model → Prediction
            (数据量决定模型质量)

因果推断:  Causal Graph + Data → Identification → Estimation
           ↑ 这是关键                 ↑ 这是辅助

一个正确指定的因果图 + 几百个样本，
远好于错误假设 + 百万样本。

═══════════════════════════════════════════════════════════════
二、三类必需数据
═══════════════════════════════════════════════════════════════

2.1 因果结构知识 (Causal Graph / DAG)
    来源                        规模要求
    ─────────────────────────────────────
    领域专家标注                 每条因果边 = 1 条专家判断
    因果发现算法 (PC, FCI, GES)  取决于变量数 n:
                                  PC/FCI: O(n²) 条件独立性检验
                                  每个检验需要 ~100-1000 样本
    文献知识图谱                 已有医学/经济学数据库
    自然实验/工具变量            需识别至少 1 个外生变量
    混合：专家 + 算法             最实用方案

2.2 观测数据 (Observational Data)
    用途：估算结构方程的参数，或直接估计因果效应
    规模：
      线性 SCM:           200-2000 样本 (取决于变量数和噪声)
      非参数 (匹配/加权):  500-10000 样本
      深度模型 (DML等):    5000-50000 样本
      IV (工具变量):       500-5000 样本 (需强工具变量)

    格式：每行一个样本，每列一个变量 (干净的表格数据)
    关键要求：
      ✓ 无系统性缺失
      ✓ 变量定义清晰 (SES 是什么意思？如何量化的？)
      ✗ 不需要标签 (y)——因为目标变量本身就是观测变量

2.3 实验/干预数据 (Experimental Data)
    用途：验证因果效应，校准模型
    规模：即使几十到几百个样本也极有价值
    格式： (干预前状态, 干预值, 干预后结果)
    来源：A/B 测试、RCT、自然实验 (DID, RDD)、
          工具变量 (IV)

═══════════════════════════════════════════════════════════════
三、数据集构建方法
═══════════════════════════════════════════════════════════════

3.1 从零构建 (针对特定领域)

Step 1: 变量定义
  → 确定内生变量 (观察到的) 和外生变量 (未观察的/噪声)
  → 每个变量必须有明确的量化定义
  → 例: "收入" = 年收入 (万元), "教育" = 受教育年限

Step 2: 因果图构建
  → 专家访谈: "你认为 X 影响 Y 吗？" → 边 X→Y
  → 或使用因果发现算法从数据中学习:
     - PC 算法: 基于条件独立检验
     - GES 算法: 基于 BIC 评分贪婪搜索
     - LiNGAM: 基于独立成分分析 (线性非高斯)
  → 标记潜在的未观测混淆因子 (U)

Step 3: 可识别性检查
  → 对于每个目标查询 P(Y|do(X)):
     - 运行 back-door 检查 → 找到调整集
     - 如果不可识别，尝试 front-door 或 IV
     - 完全不可识别 → 标记为需要实验数据

Step 4: 数据收集
  → 根据可识别性结果，收集调整集中所有变量的数据
  → 如果 IV 识别，确保 IV 满足三个条件
     (相关性、排他性、外生性)
  → 收集足量样本 (至少 200+，越多越好)

Step 5: 效应估计
  → 线性模型: 回归 + 调整集
  → 非参数: 倾向得分匹配 (PSM)、逆概率加权 (IPW)
  → 机器学习: 双重机器学习 (DML)、因果森林
  → 敏感性分析: 评估未观测混淆的影响

3.2 利用现有数据集 (快速起步)

推荐公开数据集:
  • IHDP (Infant Health and Development Program)
    → 经典的因果推断基准，747 样本，25 协变量
  • Jobs (LaLonde, 1986)
    → 职业培训的因果效应，RCT + 观测对照
  • ACIC (Atlantic Causal Inference Conference)
    → 每年更新的模拟数据集，已知真实效应
  • Twins (Almond et al., 2005)
    → 双胞胎出生体重与婴儿死亡率
  • MIMIC-III (医疗)
    → ICU 患者数据，丰富的治疗-结局变量
  • 经济学: CPS, PSID, NLSY
    → 大规模社会经济面板数据

═══════════════════════════════════════════════════════════════
四、与 LLM 结合：减少数据需求
═══════════════════════════════════════════════════════════════

如果你将 LLM 集成到智能体中，可以用 LLM 来：

1. 因果图提议 (Causal Graph Proposal)
   → 输入: "Does education affect income?"
   → LLM 输出: 可能的因果图 (含混杂因子)
   → 人类验证 + 算法精炼

2. 变量解释 (Variable Definition)
   → 输入: "SES 是什么？如何衡量？"
   → LLM 输出: 定义 + 可能的代理变量

3. 文献检索 (Literature Mining)
   → 从论文中自动提取已发表的因果效应估计
   → 构建因果知识图谱

这种方式可以大幅减少对领域专家的依赖，
将数据需求从「未知」降到「只需估计参数」。

═══════════════════════════════════════════════════════════════
五、具体规模建议 (针对当前智能体)
═══════════════════════════════════════════════════════════════

当前智能体阶段 (DAG + back-door/front-door/IV 识别):

场景                   最少数据       推荐数据       数据类型
──────────────────────────────────────────────────────────
线性 SCM (小规模)      200 样本       1000 样本      观测数据
线性 SCM + 混杂        500 样本       2000 样本      观测 + 调整集变量
非参数 (PSM/IPW)       1000 样本      5000 样本      观测 + 协变量
工具变量 (IV)          500 样本       3000 样本      需要有效 IV
因果发现 (PC算法)      200/变量对     1000/变量对    观测数据
双重机器学习 (DML)     2000 样本      10000 样本     观测 + 协变量

关键结论:
  数据量 >> 变量数 × 10  (粗略经验法则)
  50 个变量 → 至少 500 样本
  结构知识比数据量更重要
  一个小规模的 RCT 比大海的观测数据更有价值
