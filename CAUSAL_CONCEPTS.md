## 一、量子力学中的因果

### 1.1 因果没有消失，而是从确定性变成了概率性

```
经典物理 (拉普拉斯):              量子力学:
                                  
  X → Y (确定性)                   X → Y (概率性)
  给定 X，Y 一定发生                给定 X，Y 的分布改变了
                                  
  推球 → 球滚出去                   测量自旋 → 50% 向上, 50% 向下
  原因完全决定结果                  原因只决定结果的概率分布
```

**关键点**：概率分布的改变就是因果效应。量子力学没有消灭因果，它把因果从"$X$ 导致 $Y$"变成了"$X$ 改变了 $Y$ 的概率"。

### 1.2 SCM 中的 ε 就是为量子力学预留的

```
Y = f(X, ε)

这个 ε 的三重含义:
  经典:  ε = 测量误差         (我们不知道，但物理上是确定的)
  统计:  ε = 样本随机性       (抽样带来的，大样本下消失)
  量子:  ε = 内在不确定性     (物理上就不确定，即使完全知道状态)
```

Agent 的 do-calculus、反事实推理、ATE —— 全部适用于概率性因果：

$$P(Y \mid do(X=1)) \neq P(Y \mid do(X=0))$$

即使每次测量结果不同，只要 $do(X)$ 改变了 $Y$ 的分布，因果效应就存在。

### 1.3 Bell 定理挑战的不是因果，是定域性

```
经典直觉:  A ← hidden → B     (共同原因解释一切)

量子现实:  A ⟷ B              (不通过任何局部变量的直接关联)

这在因果图里表示为:
  A ↔ B  (confounded — FCI 算法的 ↔ 边类型)
```

FCI 算法输出 PAG 中的 `↔` 边——它的本意是"存在未测量的混淆变量"，但它恰好也是量子纠缠在因果图中的最佳表示。

### 1.4 对 Agent 的影响

```
零影响。

Pearl 设计 SCM 时就考虑了概率性因果：
- do(X) 仍然改变 Y 的分布
- 反事实仍然有意义 ("如果测的是 x 方向而非 z 方向？")
- ATE 仍然可定义和估计

那个 ε 不是 bug，是 feature。
```

## 二、人类创新与因果推断

### 2.1 创新 = 在反事实层操作，带回一个干预方案

Pearl 的因果阶梯：

```
阶梯 1 — 关联 (Association):
  "鸟有翅膀 → 鸟能飞"
  这是观察，不是创新。所有动物都能做到。

阶梯 2 — 干预 (Intervention):
  "如果我给自己装上翅膀会怎样？"
  古代有人试过——摔死了。干预能验证，但不能创新。

阶梯 3 — 反事实 (Counterfactual):
  "如果空气流过固定不动的翅膀……"
  "如果翅膀形状是弧面而非平面……"
  "如果推力不由拍打产生，而由旋转产生……"
  
  这些场景从未在自然界中出现过。
  它们只存在于"如果……会怎样"的想象中。
  莱特兄弟的飞机，诞生于这一层。
```

### 2.2 创新 = 在因果图空间中搜索

```
普通反事实:
  "如果我昨天早点睡，今天就不会困"
  只改变一个变量，在一个已知的因果模型里

创新反事实:
  "如果摩擦不是阻碍而是推进力会怎样？"
  改变因果结构本身，重新定义关系 → 产生了轮子
  
  "如果信息可以脱离物理载体而传播？"
  → 产生了文字、印刷术、互联网
  
  "如果计算可以不通过人类思维？"
  → 产生了计算机
```

### 2.3 创新的五步因果循环

```
1. 在已有的 SCM 中执行反事实推理
   "如果这个变量不是 X 而是 Y，结果会怎样？"

2. 发现当前 SCM 的结构本身限制了解的空间
   "我们的因果图假设 A→B→C，但如果 A→C 直接呢？"

3. 大胆修改因果图本身
   "让我加一条从未观察过的边"
   "让我删掉一条被认为理所当然的边"
   "让我反转因果方向"

4. 在修改后的因果图上运行 do-calculus
   "如果这个新边存在，干预 A 会产生什么效应？"

5. 如果效应是理想的 → 去现实世界做实验验证
   如果效应不存在 → 回到步骤 3，再改
```

### 2.4 Agent 作为创新引擎的能力边界

```
能做的:
  ✓ 在给定因果图中做反事实推理
  ✓ 量化干预的效果
  ✓ 告诉你"要达到目标 Y，你应该干预哪个 X"
  ✓ 用 do-calculus 在所有可能调整集中搜索最优解

不能做的:
  ✗ 自动产生新的因果图
  ✗ 想象一个从未被观测过的变量
  ✗ 问"如果因果方向反过来会怎样？"
  ✗ 在不同领域间做因果结构类比
```

## 三、LLM 模式下的分工

### 3.1 五步管线中的职责划分

```
Step 1:  自然语言 → JSON 因果图    │  LLM 做  (语言→结构)
Step 2:  JSON → CausalDAG          │  Agent 做 (数据结构)
Step 3:  DAG → 识别效应             │  Agent 做 (do-calculus)
Step 4:  数据 → 诊断 → 估计 ATE     │  Agent 做 (统计算法)
Step 5:  数字 → 中文解释             │  LLM 做  (结构→语言)
```

**LLM 只管两端：输入翻译和输出翻译。中间的因果推理全是 Agent 的算法在做。**

### 3.2 为什么这个分工重要

```
如果 LLM 做了因果推断:
  你问 "吸烟导致肺癌吗"
  LLM 从训练数据中回忆 "是的，研究表明相关系数 0.8"
  → 那是关联，不是因果
  → LLM 不知道 P(Y|do(X)) 和 P(Y|X) 的区别

现在 Agent 的做法:
  LLM:  "这句话里有变量 Smoking, LungCancer, Genetics..."
  Agent: "Genetics 是混杂，需要 back-door 调整"
  Agent: "在 G_{X̄} 中，调整 Genetics 后 ATE = ..."
  LLM:  "用中文告诉你：控制了遗传因素后，吸烟的因果效应是..."
  
  → LLM 不知道 back-door 是什么
  → Agent 不知道 "吸烟" 的中文含义
  → 各司其职，组合起来才完整
```

### 3.3 可验证的证据

Agent 输出中的 Step 3 显示：
```
Method: Back-door adjustment
Adjustment set: {FamilyBackground, Ability}
```

这是 `core/identification.py` 里的 `identify_effect()` 算出来的——纯图论算法，和 LLM 无关。

如果 LLM 在做因果推断，它会输出类似"根据我的训练数据，教育和收入正相关"——不会有 "back-door adjustment" 这个概念。

### 3.4 一句话

**Agent 是大脑（推理），LLM 是耳朵和嘴巴（听和说）。**

## 四、反事实推理深度解析

### 4.1 反事实不需要大量数据

```
任务              需要什么              数据量要求
─────────────────────────────────────────────────
因果发现 (PC/GES)   观测数据                500-2000 样本
效应估计 (ATE)      观测数据 + 调整集       300+ 样本
敏感性分析          估计结果                不需要额外数据
反事实推理          SCM (结构方程)          1 条观测 + 已知方程
```

反事实只需要 **1 条观测** + **已知的结构方程**。

### 4.2 三步反事实

```
Step 1: Abduction (溯因) — 反推噪声

  从观测反推这个人的独有特质 ε:
  ε = Y_obs - f(parents_obs)
  
  张三: Smoking=2.0, Cancer=3.0
  ε = 3.0 - (0.5×1.8 + 0.2×2.0 + 0.4×1.0) = 1.3
  
  这个 ε 不是"误差"——它是张三的身份。

Step 2: Action (行动) — 施加干预

  do(Smoking=0) → 修改方程:
  Tar_cf     = 0.8 × 0 + ε_T  = 0.2
  Cancer_cf  = 0.5 × 0.2 + 0.2 × 0 + 0.4 × 1.0 + 1.3 = 1.8

Step 3: Prediction (预测) — 计算新结局

  如果张三不吸烟: Cancer = 1.8
  实际吸烟:       Cancer = 3.0
```

### 4.3 ε 是关键——对"事实没发生怎么知道"的回答

```
ε 不是模型的"误差"。
ε 是这个人不可观测的、稳定的个体特质。

张三戒烟后:
  他的基因还是他的基因 ✓
  他对焦油的代谢方式不变 ✓
  他的其他生理特征不变 ✓
  
  改变的是: Smoking 从 2.0 → 0
  ε 没有变 —— 因为张三还是张三

"没发生的事实"不需要被观测。
它被推导出来，通过:
  1. 已知的因果方程
  2. 这个人的所有其他信息
  3. 他的独有特质 ε
```

### 4.4 为什么机器学习做不到

```
机器学习:
  输入: 1000 个 (学历=16, 收入=8万) 和 1000 个 (学历=12, 收入=5万)
  预测: "学历=16 → 收入约 8万"
  
  问: "张三学历=16, 收入=8万。如果学历=12, 收入多少？"
  ML: "学历=12 的人平均收入 5万 → 张三会是 5万"
  
  问题: 这个 5 万是所有学历=12 的人的平均，
        不是 "张三" 这个人。
        它没有保留 "张三独有的 ε"。

SCM:
  "张三是张三——他的 ε 不变。
   只是学历从 16 变成 12。
   如果因果方程 f(Education=12) + ε 是多少？
   答案是那个值——不管别人是多少。"
```

### 4.5 反事实的准确性：三道防线

```
防线 1: SCM 在群体层面可验证

  个体反事实无法验证，但群体干预可以：
  如果 SCM 正确预测了随机实验的 ATE，
  那它在个体反事实上也遵循同样的因果机制。

防线 2: 敏感性分析

  E-value 告诉你 "需要多强的未建模因素才能推翻结论"
  如果 E-value=8.0:
  "即使 SCM 的系数偏差 8 倍，结论方向不变"
  → 反事实虽然不是精确的，但方向是确定的

防线 3: 物理定律约束

  F=ma → a 的反事实 = F/m 是精确的
  你的 Agent 的 PhysicsInformedSCM 已经实现了这一点

结论:
  完全精确的反事实:         不可能 (需要观测平行宇宙)
  高度可信的反事实:         物理定律 + 验证过的 SCM + 大 E-value
  方向性反事实 (最常用):     即使不知道精确值，知道方向就够决策了
```

## 五、微分方程与因果

### 5.1 微分方程是因果的数学原语

```
因果图:    X ──→ Y
          意思是 "X 的改变会导致 Y 的改变"

微分方程:  dY/dt = f(X)
          意思是 "Y 的变化率由 X 决定"
          
二者说的是同一件事: X 是 Y 的原因。
微分方程多了一层——它描述了因果效应的瞬时动态。
```

### 5.2 从微分方程到因果图

```
微分方程系统:
  dX/dt = aX + bY        →  X 被自己和 Y 影响
  dY/dt = cX              →  Y 只被 X 影响

对应的因果图:
  Y ──→ X
  └────┘
  
  (Y 影响 X 的变化率，X 也影响 Y 的变化率)

因果图丢失了时间导数信息（变化多快），但保留了因果方向。
```

### 5.3 do-calculus 和微分方程中的"干预"是同一概念

```
微分方程中的干预:
  dY/dt = f(X)     —— 自然演化
  dY/dt = f(x_0)   —— 受控系统
  
  这就是 do(X=x_0)！

因果图中的干预:
  do(X=x₀) → 删除所有指向 X 的边 → 只保留 X 向外的箭头
  
  微分方程中已经完全实现了这一点——只是用不同的语言。
```

### 5.4 Agent 中微分方程与因果图的结合

```
单摆的例子:

  微分方程:  θ̈ + (g/l)·sinθ = 0

  因果图:    Length ──→ Period
             Gravity ──→ Period

  干预分析:
    do(Length=2m) → 新方程: θ̈ + (g/2)·sinθ = 0
    → 周期变长 √2 倍

  Agent 的操作:
    scm.intervene({"Length": 2.0})
    ActionPrinciple 验证新方程下的轨迹
```

### 5.5 SCM 方程就是离散化的微分方程

```
连续时间:  dY/dt = β·X(t)

离散时间:  Y_{t+1} = Y_t + β·X_t · Δt
           = (1)·Y_t + (β·Δt)·X_t
           
           这就是一个 SCM！
           Y_t 和 X_t 是 Y_{t+1} 的原因。

SCM 方程:  Y = β·X + ε
           ← dY/dt = β·X 的离散版
```

### 5.6 各自的长处

```
微分方程:
  ✓ 精确描述动态 (变化率、加速度、振荡)
  ✓ 可以计算整条轨迹
  ✗ 没有显式区分 "观测" 和 "干预"
  ✗ 没有 do-calculus 来做非参数因果识别

因果图:
  ✓ 显式区分 P(Y|X) 和 P(Y|do(X))
  ✓ do-calculus 可以在任何 DAG 上操作
  ✗ 不描述时间动态 (只关心 "谁影响谁")

结合 (Agent 的做法):
  微分方程提供动态 → SCM 捕获因果方向 → do-calculus 推理干预
  physics.py (微分方程) + scm.py (因果图) = 完整的物理因果 Agent
```
