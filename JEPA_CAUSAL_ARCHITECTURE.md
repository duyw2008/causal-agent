# JEPA × Causal Agent 对比分析与合成架构

> 版本: v0.9.9 | 日期: 2026-05-18
> 状态: 架构设计讨论

---

## 一、定义

### 1.1 Causal Agent (因果推断智能体)

```
Causal Agent 是一个基于结构因果模型 (SCM) 和 do-calculus 的推理引擎。
它不依赖神经网络训练来进行因果推断——推理能力来自图论算法、条件
独立性检验和统计估计，而非数据驱动的模式匹配。

核心组件:
  - CausalDAG:   有向无环图，编码变量间的因果方向
  - SCM:         结构方程，编码因果强度 (f: PA(X) → X)
  - do-calculus: 符号规则，区分 P(Y|X) 与 P(Y|do(X))
  - CI tests:    统计检验，从数据中发现因果结构

本质: 一种可解释的、基于数学定理的推理系统
```

### 1.2 JEPA (联合嵌入预测架构)

```
JEPA (Joint Embedding Predictive Architecture) 是 LeCun 提出的自监督
学习框架。核心思想：不在原始信号空间做预测 (像素级)，而在学习到的
隐空间 (嵌入空间) 做预测。

核心组件:
  - Encoder:      x → s      (原始信号 → 嵌入向量)
  - Predictor:   s_x → s_y  (隐空间条件预测)
  - Energy:     E(s_x, s_y)  (衡量两个嵌入的兼容性)

本质: 一种样本高效的感知层表征学习方法
```

### 1.3 根本区别

```
Causal Agent 解决 "为什么" 和 "如果……会怎样"
JEPA 解决 "世界是什么样的"

前者是推理层，后者是感知层。不竞争，可互补。
```

## 二、功能

### 2.1 Causal Agent 能做什么

| 功能 | 实现 | 输入 | 输出 |
|------|------|------|------|
| 因果图构建 | `CausalDAG` | 变量名 + 边 | 有向无环图 |
| d-separation 判断 | `is_d_separated()` | 三个变量集 | True/False |
| 因果发现 (PC) | `pc_algorithm()` | 数据 + α | CausalDAG (CPDAG) |
| 因果发现 (FCI) | `fci_algorithm()` | 数据 + α | PAG (含隐变量标记) |
| 因果发现 (GES) | `ges_algorithm()` | 数据 | CausalDAG |
| 效应识别 (back-door) | `find_back_door_adjustment()` | DAG + T + Y | 调整集 Z |
| 效应识别 (front-door) | `find_front_door_adjustment()` | DAG + T + Y | 中介 M + 调整集 |
| 效应识别 (IV) | `check_instrument()` | DAG + T + Y + Z | 工具变量有效性 |
| do-calculus 规则 | `do_calculus_rule1/2/3()` | DAG + 变量集 | 规则是否适用 |
| 效应估计 (11 种) | `estimate_effect()` | 数据 + T + Y + Z | ATE ± 95% CI |
| 反事实推理 | `SCM.counterfactual()` | 观测 + 干预 + 目标 | 反事实值 |
| 敏感性分析 | `full_sensitivity_report()` | 估计结果 + 数据 | Γ临界值 + E-value |
| 物理规律约束 | `PhysicsInformedSCM` | DAG + 物理定律 | 约束后的 SCM |
| 时间序列因果 | `granger_causality_test()` | 多变量时序 | p-value |

### 2.2 JEPA 能做什么

| 功能 | 实现 | 输入 | 输出 |
|------|------|------|------|
| 感知压缩 | Encoder | 图像/视频/音频 | 嵌入向量 s |
| 隐空间预测 | Predictor | s_x | ŝ_y |
| 兼容性度量 | Energy Function | (s_x, s_y) | E ∈ ℝ |
| 自监督预训练 | 整体架构 | 无标签数据 | 通用 Encoder |

### 2.3 Causal Agent 不能做什么

```
✗ 处理高维原始感知输入 (图像、视频、音频)
  - DAG 的顶点必须是已命名的抽象变量
  - 无法从像素中直接构建因果图

✗ 非线性条件独立性检验 (仅 Fisher z — 线性高斯假设)
  - 非线性依赖会被漏掉或误判

✗ 跨模态因果推理
  - "图像中的病灶特征"与"文本病历的症状"之间无法自动建立因果连接

✗ 从无标签数据中自动学习变量表示
  - 变量必须由人类预先定义
```

### 2.4 JEPA 不能做什么

```
✗ 因果推断
  - 不区分 P(Y|X) 和 P(Y|do(X))
  - 不知道哪些关联在干预下会断裂
  - 预测器只学习观测条件分布，不是干预分布

✗ 可解释推理
  - 嵌入向量 s 是黑箱
  - 无法给出类似 "控制 Z 后，ATE=2.3" 的可解释结论
  - 无法告诉人类 "为什么"

✗ 敏感性分析
  - 没有 Rosenbaum bounds
  - 不能回答 "未观测混淆需要多强才能推翻结论？"

✗ 结构化因果表达式
  - 没有 do-calculus 符号推理能力
  - 无法推导 P(y|do(x)) = Σ_z P(y|x,z)P(z)
```

## 三、解决的问题

### 3.1 两类问题的明确划分

```
┌─────────────────────────────────────────────────────────┐
│  Causal Agent 解决的问题 (推理层)                       │
│                                                         │
│  Q1: "教育年限对收入的因果效应是多少？"                  │
│  Q2: "如果我用药，康复率会改变多少？(do-calculus)"       │
│  Q3: "如果这个人当年没上大学，收入会是多少？(反事实)"    │
│  Q4: "观测到的相关性中，有多少是因果、多少是混淆？"     │
│  Q5: "这个效应估计对未测量的混杂因素有多敏感？"          │
│                                                         │
│  特点: 抽象语义变量 + 干预性推理 + 可解释输出            │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│  JEPA 解决的问题 (感知层)                                │
│                                                         │
│  Q1: "给定这张 MRI 影像的局部, 整体结构应该是什么样的？" │
│  Q2: "两段视频片段在内容上是连续的吗？"                  │
│  Q3: "这个物体在视角变化后, 嵌入向量是否不变？"          │
│  Q4: "如何在没有标签的情况下学到有用的视觉表示？"        │
│                                                         │
│  特点: 高维感知输入 + 预测性推理 + 隐空间输出             │
└─────────────────────────────────────────────────────────┘
```

### 3.2 交叉地带：两者各自无法独立解决、但合成后可以

```
问题: "根据这组病人的MRI影像和血液指标, 
        哪种治疗方案最可能延长生存期？原因是什么？"

Causal Agent 单独:
  ✗ 无法读取 MRI 影像
  ✗ 只能处理 "肿瘤大小" "血压" 等已提取的标量特征

JEPA 单独:
  ✗ 能编码 MRI → 嵌入, 但不知道嵌入的哪些维度对应 "治疗" 的因果效应
  ✗ 无法区分 "相关性" 和 "因果效应"
  ✗ 无法给出 "原因是什么" 的解释

Causal + JEPA 合成:
  ✓ JEPA Encoder: MRI → 因果相关嵌入
  ✓ Causal Agent: 嵌入 + 血液指标 → DAG → 识别 → 估计
  ✓ 输出: "治疗方案 A 的 ATE=+8.3个月, 方案 B 的 ATE=+2.1个月,
          选择方案 A。原因: do-calculus 显示方案A通过缩小肿瘤
          的因果路径 (中介效应) 而非伴随症状路径。"
```

## 四、能力边界

### 4.1 Causal Agent 的能力边界

```
✅ 能做:
  - 标量表格数据的全链路因果推断 (发现→识别→估计→反事实→敏感性)
  - 可解释的因果效应估计 (带置信区间)
  - 显式建模隐变量混淆 (FCI)
  - 物理定律作为因果约束
  - 人工构建的因果场景

⚠️ 边界:
  - 变量必须是人类定义的标量语义变量
  - CI 检验假设线性高斯 (Fisher z)
  - 高维数据 (图像/视频/音频) 无法直接输入
  - do-calculus 集成不完全 (R1-R3 存在但未接入 identify_effect 管道)
  - 无中介分析 (NDE/NIE)
  - 无因果表示学习 (Causal Representation Learning)

❌ 不能做:
  - 从原始感知信号中发现因果结构
  - 端到端的多模态因果推理
  - 非线性 CI 检验 (当前仅 Fisher z)
```

### 4.2 JEPA 的能力边界

```
✅ 能做:
  - 高维感知信号的嵌入学习 (图像/视频/音频)
  - 隐空间的预测 (比像素空间样本效率高 ~30x)
  - 自监督预训练 (无需标签)
  - 通用表示学习 (Encoder 可跨任务复用)

⚠️ 边界:
  - 只预测隐空间, 不生成像素 (简化版, 但放弃了生成能力)
  - 预测的是观测分布 P(Y|X), 不是干预分布 P(Y|do(X))
  - 嵌入缺乏可解释性
  - 训练稳定性依赖于嵌入空间的维度选择
  - 缺乏理论保证的统计校准

❌ 不能做:
  - 因果推断 (do-calculus, 反事实, 敏感性)
  - 可解释推理 (无法解释 "为什么")
  - 符号推理 (无法处理抽象逻辑规则)
  - 显式建模变量间的因果方向
  - 区分混淆关联与因果效应
```

### 4.3 边界可视化

```
问题复杂度
    ▲
    │                              ┌─────────────────┐
    │    "从MRI影像和血液指标       │  JEPA + Causal  │
    │     判断最优治疗方案"         │  (合成)         │
    │                              └─────────────────┘
    │
    │  ┌─────────────────┐         ┌─────────────────┐
    │  │ Causal Agent    │         │  (空白)         │
    │  │ (表格因果推断)  │         │                 │
    │  └─────────────────┘         └─────────────────┘
    │
    │  ┌─────────────────┐         ┌─────────────────┐
    │  │ JEPA            │         │  (空白)         │
    │  │ (图像补全/匹配) │         │                 │
    │  └─────────────────┘         └─────────────────┘
    │
    └──────────────────────────────────────────────►
                      数据维度

左边: 低维表格数据          右边: 高维感知数据
      → Causal 和 JEPA       → 只有合成才能覆盖
        各自独立可用
```

## 五、训练数据需求

### 5.1 数据需求量级对比

```
方法                     数据需求量级          典型任务
────────────────────────────────────────────────────────────
Diffusion (图像生成)     10⁶-10⁹  图片         文生图
LLM (自回归)             10⁹-10¹² tokens       对话
JEPA (编码器训练)        10⁴-10⁶  样本         图像理解
JEPA (预测器+能量)       10³-10⁴  样本对       回归/度量
────────────────────────────────────────────────────────────
Fisher z-test            ~200   样本/变量对     CI 检验
PC 算法                  ~500   样本/图        因果发现
GES (BIC)                ~1000  样本/图        因果发现
SCM 拟合                 ~500   样本/图        效应估计
```

### 5.2 JEPA 的数据效率优势（相对于谁）

```
JEPA vs 传统生成模型:
  - 预测隐空间而非原始像素 → 样本效率高 ~30 倍
  - 不需要建模无关细节 → 每个样本的信息利用率更高
  - LeCun 的直觉: 1000维像素预测 → 32维嵌入预测 ≈ 30x 加速

JEPA vs Causal Agent 的 CI 检验:
  - JEPA 需要训练 (10³-10⁴ 样本)
  - Fisher z-test 不需要训练 (闭式解, ~200 样本)
  - 在标量变量场景下: CI 检验的数据效率 > JEPA 约 10-100 倍
  - 但在图像场景下: CI 检验根本不可用, JEPA 是唯一选择
```

### 5.3 现有 causal_agent 数据集对 JEPA 的充分性

```
type1 数据集: 100 DAG × 500 样本 = 50,000 数据点

JEPA Encoder 训练:
  - 标量输入: 无感知压缩需求 → 恒等映射即可
  - 高维输入: 需要 10⁵-10⁶ 样本 → type1 完全不够 (且 type1 不是高维数据)
  结论: type1 对 Encoder 训练不够，但对当前场景也不需要

JEPA Predictor 训练:
  - 3 层 MLP, 标量对 → 50,000 样本
  结论: 足够 ✓

JEPA Energy 训练:
  - 余弦距离/点积, 50,000 正负对
  结论: 足够 ✓

总体:
  - 在你的标量变量场景下, Predictor 和 Energy 可以用 type1 训练
  - 但 Encoder 没有存在价值 (标量本身就是最佳嵌入)
  - 如果你将来处理图像数据, type1 完全不能用于 Encoder 预训练
    → 需要另备大规模图像数据集 (如 ImageNet)
```

### 5.4 不需要大量数据的 JEPA 用法

```
JEPA Predictor 单独使用 (跳过 Encoder 预训练):

  适用场景: 变量已经是抽象语义标量
  数据需求: 与你的 type1 数据集匹配 (5 万样本)
  训练难度: 低 (3 层 MLP, 分钟级训练)
  产出: 一个隐空间预测器, 可作为 PC 算法的增强信号

  限制: 不提供 Encoder 的感知压缩提升
        没有 "从图像中发现因果" 的能力
        本质上是"用神经网络替代 Fisher z" — 收益有限
```

## 六、加入 JEPA 后的功能变化

### 6.1 不增强的能力（JEPA 无增益）

```
因果推断精度:
  - d-separation: JEPA 嵌入不如偏相关精确
  - do-calculus: 纯符号推理, JEPA 不参与
  - 反事实: SCM 已经精确, JEPA 的隐空间预测不带来额外精度
  - 敏感性: Rosenbaum + E-value 基于统计量, JEPA 不适用
  - 效应估计: OLS/IPW/DR 是解析方法, JEPA 无法改进

结论: 在 "因果推断本身" 的任务上,
      加 JEPA = 加误差 + 加复杂度 + 不加精度
```

### 6.2 新增的能力（质变）

```
能力 1: 高维数据 → 因果变量 (从不能做到能做) ★★★★★
  
  现状: 人工从图像/视频/文本中提取标量特征
        耗时数天到数周, 依赖领域专家
        提取什么特征取决于人的先验, 可能遗漏关键信息

  +JEPA: Encoder 自动压缩感知信号为嵌入向量
        训练几小时, 数据驱动, 减少人为偏差
        嵌入可以捕捉人类可能忽略的微妙模式

  场景示例:
    MRI 影像 → JEPA Encoder → 嵌入 → CausalDAG → do-calculus → ATE
    (当前完全不可行)


能力 2: 非线性条件独立性检验 (从弱到中) ★★★

  现状: Fisher z-test 假设线性高斯
        非线性依赖可能被漏掉

  +JEPA: 能量函数 E(s_X, s_Y|s_Z) 无分布假设
        可以捕捉任意条件依赖

  代价: 能量函数的统计校准不确定
        Type I error 无理论保证 (Fisher z 有解析 p-value)
        需要额外的 permutation test 来校准


能力 3: 隐空间反事实 (新增) ★★★★

  现状: 反事实仅在原始变量空间
        do(X=0) → 重算 SCM → Y_cf
        无法在图像/文本等非结构化数据上执行

  +JEPA: 反事实在隐空间执行
        do(嵌入_病灶=缩小) → JEPA Predictor → 新嵌入 → (可选)解码
        图像编辑式反事实: "如果病灶缩小一半, 生存期预测会怎样?"

  代价: 可解释性丧失
        "把 s_3 调到 0.7" 无法翻译成人类可理解的操作
        解码回像素空间需要额外训练 Decoder（增加复杂度）


能力 4: 跨模态 / 跨任务迁移 (新增) ★★★

  现状: 每个领域从零构建因果图
        医学影像的因果发现经验不能迁移到自动驾驶

  +JEPA: 大规模无标签数据上预训练 Encoder
        → 同一个 Encoder 用于多个下游因果推理任务
        (类似于 BERT 在 NLP: 预训练 → 下游因果任务微调)

  代价: 需要预训练语料库 + GPU 资源
        预训练与因果微调之间的分布偏移可能损害因果推断精度


能力 5: 隐空间因果发现 (前沿) ★★

  现状: causal discovery 在原始变量空间运行
        (变量已命名, 标量, 线性假设)

  +JEPA: 因果发现在隐空间运行
        嵌入维度可能比原始变量更精炼
        非线性因果结构可能被捕捉

  代价: 高风险研究方向
        "隐空间的边" 如何验证？无 ground truth
        隐空间方向与原始因果方向不一定对齐
```

### 6.3 收益矩阵

```
场景                         不加 JEPA   加 JEPA    收益   建议
────────────────────────────────────────────────────────────
表格数据因果推断              完全胜任    无变化      0    不要加
────────────────────────────────────────────────────────────
图像/视频因果推理             完全不行    可行      极高   优先做
────────────────────────────────────────────────────────────
文本病历的因果分析             部分(规则)  可行      高     做
────────────────────────────────────────────────────────────
传感器时序的因果发现            部分(存在)  增强      中     可选
────────────────────────────────────────────────────────────
多模态(图像+表格)因果推理      完全不行    可行      极高   最前沿
────────────────────────────────────────────────────────────
非线性因果发现                 部分漏掉    增强      中     先确认数据
                                                            是否真违反
                                                            线性假设
────────────────────────────────────────────────────────────
```

### 6.4 一句话总结

```
JEPA 不增强因果推理精度,
但让因果推理能处理之前完全处理不了的数据类型。

如果你的变量已经是标量:
  JEPA 无增益 — 不要加。

如果你想让 agent 看 MRI 影像做因果推断:
  JEPA 是唯一的路 — 必须加。
```

## 七、合成架构设计

### 7.1 分层智能体（不是两个 agent 对话）

```
关键设计原则: 不是两个 agent 对话，是一个 agent 两层架构。

┌──────────────────────────────────────────┐
│          Causal Agent (推理层)            │
│  ┌─────────┐  ┌──────┐  ┌─────────────┐  │
│  │ CausalDAG│  │ SCM  │  │ do-calculus │  │
│  └────┬─────┘  └──┬───┘  └──────┬──────┘  │
│       │           │             │         │
│       └───────────┴─────────────┘         │
│                   │                       │
│          因果变量 (抽象语义)               │
│        "教育年限" "收入" "健康状态"        │
├──────────────────────────────────────────┤
│                   ▲                       │
│                   │ 隐空间 ↕ 语义空间      │
│                   │ 映射层                │
│                   ▼                       │
│          JEPA (感知层)                     │
│  ┌────────┐  ┌──────────┐  ┌──────────┐  │
│  │Encoder │→ │Predictor │→ │  Energy  │  │
│  └────────┘  └──────────┘  └──────────┘  │
│                   │                       │
│            原始感知输入                    │
│       "图像" "时序" "文本" "传感器"        │
└──────────────────────────────────────────┘
```

### 7.2 三个 Phase 的实施路线

```
Phase 1: 在 causal_agent 上加 JEPA 编码层
  - CausalDAG 的每个变量变成 JEPA 编码器输出的嵌入
  - 因果发现从 "X→Y" 变成 "Encoder_X → Encoder_Y"
  - 2-3 周, 用现有 type1 数据集验证, 最低风险

Phase 2: JEPA 能量函数替代 Fisher z-test
  - E(s_X, s_Y|s_Z) 替代偏相关检验
  - 需要额外做统计校准 (permutation test)
  - 3-4 周, 中等风险

Phase 3: 反事实隐空间插值
  - JEPA Predictor 在 do() 约束下做反事实预测
  - 需要额外训练 Decoder (生成式) 或直接使用嵌入级反事实
  - 4-6 周, 高风险
```

### 7.3 为什么是分层架构而非双 agent 对话

```
双 agent 对话架构的问题:

  问题 1: 训练动力学不稳定
    两个 agent 都需要学习, 相互预测对方行为
    类似 GAN 的 mode collapse — 一种 agent 可能退化输出常数

  问题 2: 信息瓶颈
    Agent A 的 causal 输出必须被压缩为 Agent B 可消费的信号
    反之亦然 → 两个 agent 之间的通信带宽限制了整体能力

  问题 3: 责任模糊
    当出错时 ("为什么预测 ATE=3.2 而非实际的 2.1?")
    不知道是 Agent A 的 DAG 错了还是 Agent B 的能量函数错了
    调试代价极高

分层架构的优势:

  优势 1: 单向信息流, 稳定
    JEPA → Causal 单向 ("感知 → 推理")
    只有一个训练目标, 可以分阶段训练

  优势 2: 可解释性不丢失
    Causal Agent 始终负责最终的解释输出
    JEPA 的隐空间只是 Causal Agent 的输入, 不是其输出

  优势 3: 模块可替换
    JEPA Encoder 可以换为其他 Backbone (CLIP, DINOv2, ViT...)
    Causal Agent 的核心算法不受影响
```

## 八、结论

### 8.1 核心命题

| 维度 | Causal Agent | JEPA | 合成 |
|------|:----------:|:----:|:----:|
| 因果推断精度 | ★★★★★ | ☆☆☆☆☆ | ★★★★★ (不变) |
| 可处理数据类型 | 标量表格 | 图像/视频/音频 | 全部 |
| 可解释性 | ★★★★★ | ☆☆☆☆☆ | ★★★★ (轻微降) |
| 数据效率 | ★★★★★ | ★★★ | ★★★★ |
| 训练难度 | 无需训练 | 中等 | 高 |
| 研究成熟度 | 成熟 | 成熟 | 前沿 |

### 8.2 什么时候做

```
现在不做:
  - causal_agent 的变量仍然是标量 → JEPA 无价值

将来做 (当数据扩展时):
  - 引入图像/视频/传感器数据时
  - Phase 1 先行 (编码层, 2-3 周)
  - Phase 2 可选 (能量 CI, 只有 Fisher z 不够时)
  - Phase 3 最后 (隐空间反事实, 最高风险)

不要做的:
  - 在标量数据上强行加 JEPA → 用更多数据换更弱的结果
  - 双 agent 对话架构 → 训练不稳定, 调试地狱
```
